🎉 Account Created!

You're all set and signed in. One important step to keep your data private:

Disable new sign-ups in Firebase so no one else can create an account on your project:

  1. Go to console.firebase.google.com and select your project
  2. In the left sidebar click Build → Authentication
  3. Click the Settings tab (not the Users tab)
  4. Under "User actions", uncheck "Enable create (sign-up)"
  5. Click Save

Your existing account and password reset will still work after this. You can find these instructions again under Settings → Data Storage.

Setup Firebase Storage

Complete all three steps below before activating your private vault.

Step 1 — Upgrade to Blaze Plan (required for Storage)

Firebase Storage is not available on the free Spark plan — it requires the Blaze (pay-as-you-go) plan. The good news: you won't be charged anything unless you exceed 5 GB storage or 1 GB/day download. For a personal private vault this is effectively free, but Firebase requires a credit card on file to enable Storage at all.

  1. Go to console.firebase.google.com
  2. Select your project
  3. In the bottom-left corner, click the Spark badge (it shows your current plan)
  4. Click Upgrade
  5. Select the Blaze plan
  6. Add a payment method and confirm
  7. You're now on Blaze — Storage is unlocked and you won't be charged unless you exceed the free limits

Step 2 — Enable Firebase Storage

  1. Still in the Firebase console, select your project
  2. Click Storage in the left nav
  3. Click Get Started
  4. Leave No cost location selected, choose region US-CENTRAL1, click Continue
  5. Choose Start in production mode (locks everything down — you'll paste the real rules in Step 3), click Create

Step 3 — Secure the Storage Rules (required)

The default rules allow any logged-in user to read anyone's files. Replace them with rules that restrict each user to their own folder.

  1. In the Storage section, click the Rules tab
  2. Replace all existing content with the rules below
  3. Click Publish
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /users/{userId}/{allPaths=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
  }
}

Step 4 — Allow the App to Download Your Files (required)

By default, Firebase Storage blocks browsers from downloading files directly. You need to run one command that tells Storage to allow your app to read files. This only needs to be done once and takes about 2 minutes. No software to install — Google Cloud has a built-in terminal you can run right in your browser.

  1. Open console.cloud.google.com in a new tab
  2. Confirm your project name appears in the top-left area next to the Google Cloud logo. If it shows a different project, click it and switch to the correct one.
  3. Find the Cloud Shell button: Look at the very top-right of the page. You'll see a row of small icons just to the left of your profile photo/avatar. Hover your mouse slowly over each icon — you're looking for one with the tooltip that says "Activate Cloud Shell". It looks like a small monitor screen. Click it.
    (If a blue banner appears asking you to authorize, click Authorize.)
  4. A terminal panel slides up from the bottom of the screen. Wait until you see a $ prompt — this can take up to 60 seconds the first time.
  5. Find your Storage bucket name: go to the Firebase console → your project → Storage (left nav) → the bucket name is shown near the top of the page. It looks like your-project.firebasestorage.app or your-project.appspot.com. Copy that name exactly.
  6. Click inside the Cloud Shell terminal, paste the command below, then press Enter:
printf '[{"origin":["*"],"method":["GET","HEAD"],"maxAgeSeconds":3600}]' > cors.json && gcloud storage buckets update gs://YOUR-BUCKET-NAME --cors-file=cors.json

You should see a line that says Updating... followed by a success message. If it asks you to authorize again, click Authorize. Once you see the $ prompt return, you're done — photos and documents will now load correctly.

📍 How Check-In Works

            


                
⚠️ How to Complete a Check-In

                
Tapping a place in the list does not save it immediately. It opens the journal entry form with the location pre-filled. You must tap Save on that form to actually record the check-in. You can add notes before saving — or just tap Save right away.


                
Finding Nearby Places

                
When the Check-In screen opens, the app automatically tries to get your GPS location and show places near you. On a good signal this takes a few seconds. On a weak signal you may see a "Could not load" message — tap Retry to try again without closing the screen.


                
Setting Your Location Manually

                
If GPS is weak or unavailable, type a city, address, or zip code in the "Search near:" field (e.g. "Bishop, GA" or "30621"). The app will find places near that location instead. The field turns green when the location is recognized.


                
Searching by Name

                
Type any part of a place name in the search box (e.g. "Golden Pantry") and results will appear from Foursquare. Each result shows the address and a distance from your current location so you can tell apart two locations with the same name. Results are biased toward where you are — if you're in Bishop, GA you'll see Bishop results, not Virginia.


                
Selecting a Place

                
Tap any result to select it. This takes you to the journal entry form with the location already attached. Add any notes you want, then tap Save to record the check-in.

export default {
  async fetch(request, env) {
    const cors = {
      'Access-Control-Allow-Origin': '*',
      'Access-Control-Allow-Methods': 'GET, OPTIONS',
    };
    if (request.method === 'OPTIONS')
      return new Response(null, { headers: cors });

    const ticker = new URL(request.url).searchParams.get('ticker');
    if (!ticker)
      return new Response(JSON.stringify({ error: 'Missing ticker' }),
        { status: 400, headers: { ...cors, 'Content-Type': 'application/json' } });

    const url = 'https://query1.finance.yahoo.com/v8/finance/chart/'
              + encodeURIComponent(ticker) + '?interval=1d&range=1d';
    try {
      const resp = await fetch(url, {
        headers: { 'User-Agent': 'Mozilla/5.0' }
      });
      const data = await resp.json();
      return new Response(JSON.stringify(data),
        { status: resp.status, headers: { ...cors, 'Content-Type': 'application/json' } });
    } catch (e) {
      return new Response(JSON.stringify({ error: e.message }),
        { status: 500, headers: { ...cors, 'Content-Type': 'application/json' } });
    }
  }
};

What it does
